# إعداد النطاق المخصص (Custom Domain)

## الخطوة 1: إنشاء ملف CNAME

1. أنشئ ملف باسم `CNAME` في المجلد الرئيسي للمشروع
2. أضف نطاقك المخصص في الملف (سطر واحد فقط):
   ```
   yourdomain.com
   ```
   أو إذا أردت استخدام www:
   ```
   www.yourdomain.com
   ```

## الخطوة 2: إعداد DNS في مقدم خدمة النطاق

### الخيار أ: استخدام CNAME (موصى به)

إذا كنت تستخدم نطاق فرعي مثل `www`:
- **النوع**: CNAME
- **الاسم**: www
- **القيمة**: `username.github.io` (استبدل username باسم المستخدم)

إذا كنت تستخدم النطاق الرئيسي (بدون www):
- **النوع**: A
- **القيمة**: 
  - `185.199.108.153`
  - `185.199.109.153`
  - `185.199.110.153`
  - `185.199.111.153`

### الخيار ب: استخدام A Records

للنطاق الرئيسي (yourdomain.com):
- **النوع**: A
- **القيمة**: أضف جميع العناوين التالية:
  ```
  185.199.108.153
  185.199.109.153
  185.199.110.153
  185.199.111.153
  ```

## الخطوة 3: تفعيل النطاق المخصص في GitHub

1. اذهب إلى المستودع على GitHub
2. اضغط على **Settings**
3. في القائمة الجانبية، اضغط على **Pages**
4. في قسم **Custom domain**، أدخل نطاقك
5. اختر **Enforce HTTPS** (سيصبح متاحًا بعد بضع دقائق)

## الخطوة 4: الانتظار

- قد يستغرق نشر تغييرات DNS من بضع دقائق إلى 24 ساعة
- تحقق من النطاق باستخدام: `nslookup yourdomain.com` أو `dig yourdomain.com`

## ملاحظات مهمة

- **لا تستخدم CNAME للنطاق الرئيسي**: استخدم A records فقط
- **استخدم CNAME للنطاقات الفرعية**: مثل www.yourdomain.com
- **تفعيل HTTPS**: بعد تفعيل النطاق، سيصبح HTTPS متاحًا تلقائيًا من GitHub
- **ملف CNAME**: يجب أن يحتوي على اسم نطاق واحد فقط في كل سطر

## أمثلة لمزودي DNS شائعين

### Cloudflare
- اذهب إلى DNS → Records
- أضف A records للنطاق الرئيسي
- أضف CNAME لـ www

### GoDaddy
- اذهب إلى DNS Management
- أضف/عدّل A records و CNAME records

### Namecheap
- اذهب إلى Advanced DNS
- أضف/عدّل Host records

## التحقق من الإعداد

بعد الانتظار، تحقق من:
1. فتح النطاق في المتصفح
2. التحقق من أن HTTPS يعمل
3. التحقق من أن الموقع يحمل بشكل صحيح

## حل المشاكل

- **النطاق لا يعمل**: تحقق من DNS propagation باستخدام [whatsmydns.net](https://www.whatsmydns.net)
- **خطأ SSL**: انتظر ساعة بعد تفعيل النطاق في GitHub
- **CNAME conflict**: تأكد من أن ملف CNAME يحتوي على نطاق واحد فقط
